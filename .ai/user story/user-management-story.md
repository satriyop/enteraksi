# user story
## user visitor/public  
- able to do registration on the website.
- able to be approved by lms admin or admin role user upon registration.
- able to get learner role automatically once approved by lms admin.
- able to login using either email address or phone number once approved.

## user with lms admin & super admin role
- able to see list of users in the User Management CRUD UI.
- able to approve user registration.
- able to create, edit, delete user, except for user with super admin role.
- able to assign/reassigned role to user : learner, trainer, content manager, event organizer, lms admin to a user.
- able to pre-approve user, newly created user (optional with button in create form).
- able to set status learner role user : active, suspended, inactive.
